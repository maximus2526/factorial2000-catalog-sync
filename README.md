# Factorial2000 Catalog Sync for Prom.ua and WooCommerce 🚀

**Factorial2000 Catalog Sync** — це легкий, перевірений часом та оптимізований плагін для WordPress, який повністю вирішує проблему синхронізації інтернет-магазинів на WooCommerce із постачальниками, що працюють через Пром. 

Він автоматично імпортує прості та варіативні товари з XML-вивантажень, контролює залишки, актуалізує ціни за розкладом та чистить базу від застарілих позицій.

> 🔥 **Важливо:** Наразі плагін знаходиться у відкритому доступі. Ви можете завантажити та протестувати базовую версію абсолютно безкоштовно. В майбутньому проект планується перевести на **платну основу** з розширеним функціоналом та преміум-підтримкою. Ті, хто підтримає проект на старті, отримають особливі бонуси!

## ⚡ Головні фічі

- **Повний імпорт з Пром:** Коректний парсинг XML-файлів, включаючи складні варіативні товари (розміри, кольори).
- **Розумне оновлення:** Автоматична актуалізація цін та статусів наявності (`instock` / `outofstock`) через WP-CRON у заданий інтервал.
- **Очищення вивантажень:** Фільтрація файлу постачальника для створення нової, "чистої" вигрузки суто з новими позиціями.
- **Сповіщення в Telegram:** Миттєві звіти у ваш чат про успішне завершення імпорту або виникнення помилок на сервері.
- **Оптимізація під Big Data:** Пакетна обробка (batch processing) та регулярне звільнення пам'яті. Легко пережовує каталоги на **10 000+ товарів** навіть на середньому хостингу. (Якщо заходить в timeout запускайте в cron)
- **Екологічність коду:** При деактивації та видаленні плагін повністю вичищає за собою всі налаштування та CRON-задачі, не засмічуючи базу даних `wp_options`.

## 📦 Встановлення

### Варіант 1. Завантаження архівом (Рекомендовано)
1. Завантажте плагін, натиснувши зелену кнопку **Code -> Download ZIP** у верхній частині цієї сторінки.
2. В адмінці WordPress перейдіть у **Плагіни -> Додати новий -> Завантажити плагін** і виберіть цей архів.
3. Активуйте плагін.

### Варіант 2. Через Git клієнт
```bash
cd wp-content/plugins/
git clone https://github.com/maximus2526/factorial2000-catalog-sync.git
```

Після цього активуйте його в панелі керування WordPress.

---

## ⚙️ Перше налаштування

1. Після активації в меню адмінки з'явиться новий розділ.
2. Перейти у вкладку **«Робота зі вигрузкою» / «Імпорт»**, вставити URL-посилання на XML від постачальника та запустити первинний імпорт.
3. У вкладці **«Оновлення»** налаштувати інтервал синхронізації цін та залишків, а також вказати Token та ID чату Telegram для сповіщень.

## 🔔 Налаштування Telegram-сповіщень

Плагін може надсилати у Telegram звіти про завершення імпорту та оновлення залишків (скільки товарів оновлено, скільки не знайдено тощо). Це **необов'язково** — якщо поля порожні, нічого не надсилається.

Щоб увімкнути сповіщення, у вкладці **«Оновлення»** треба заповнити два поля.

### 1. Telegram Token ID — токен вашого бота

Токен — це «ключ доступу» до вашого бота, через який плагін надсилає повідомлення.

1. У Telegram знайдіть бота **[@BotFather](https://t.me/BotFather)**.
2. Надішліть команду `/newbot`.
3. Вкажіть назву бота та його username (має закінчуватися на `bot`).
4. BotFather у відповідь надішле токен виду:
   ```
   1234567890:ABCdefGHIjklMNOpqrsTUVwxyz1234567890
   ```
5. Скопіюйте **весь рядок** і вставте у поле **Telegram Token ID**.

### 2. Telegram User IDs — куди надсилати

Це числовий ID чату (або кількох), у який бот надсилатиме звіти.

1. У Telegram знайдіть бота **[@userinfobot](https://t.me/userinfobot)** і натисніть **Start** — він покаже ваш числовий **Id** (наприклад `123456789`).
2. Вставте це число у поле **Telegram User IDs**.
3. Якщо отримувачів декілька — перелічіть їхні ID **через кому**: `123456789, 987654321`.

> ⚠️ **Важливо:** перш ніж бот зможе вам написати, **ви маєте першим написати своєму боту** команду `/start` (або додати бота в потрібну групу). Інакше Telegram заблокує повідомлення і сповіщення не дійде.

Для надсилання в **групу** додайте бота в групу та використайте ID групи (від'ємне число, наприклад `-1001234567890`) у полі User IDs.
